## 规范

以下是 Aleo 虚拟机 (AVM) 中每个操作码的规范。

### `abs`

[返回顶部](#标准操作码表)

#### 描述

计算输入的绝对值，检查溢出，将结果存储在目标寄存器中。

对于整数类型，添加约束以检查下溢。如果需要回绕语义，请参阅 [abs.w](#abs.w) 指令。当下溢发生在有符号整数类型的最小值时会发生此情况。例如，`abs -128i8` 会导致下溢，因为 `128` 无法表示为 `i8`。

#### 示例用法

```aleo
abs r0 into r1;
```

#### 支持的类型

| Input  | Destination |
| ------ | :---------- |
| `I8`   | `I8`        |
| `I16`  | `I16`       |
| `I32`  | `I32`       |
| `I64`  | `I64`       |
| `I128` | `I128`      |

---

### `abs.w`

[返回顶部](#标准操作码表)

#### 描述

计算输入的绝对值，在类型边界处回绕，并将结果存储在目标寄存器中。

#### 示例用法

```aleo

```

#### 支持的类型

| Input  | Destination |
| ------ | :---------- |
| `I8`   | `I8`        |
| `I16`  | `I16`       |
| `I32`  | `I32`       |
| `I64`  | `I64`       |
| `I128` | `I128`      |

### `branch.eq`

[返回顶部](#标准操作码表)

#### 描述

branch.eq r0 r1 to skip;

### `snark.verify.batch`

[返回顶部](#密码学操作码表)

#### 描述

批量验证多个 Varuna 证明，针对验证密钥数组 `first`、嵌套的公开输入数组 `second` 和单个批量证明 `third`，将布尔结果存储在 `destination` 中。此操作码**仅限 finalize 使用**，必须在 `finalize` 块内调用。在 Aleo Stack v4.6.0 中引入，作为 [ARC-0008](https://vote.aleo.org/p/arc-0008) 的一部分。

编译时限制适用：最多 32 个电路，所有电路最多 128 个实例。

#### 示例用法

```aleo
snark.verify.batch r0 r1 r2 into r3;
```

#### 操作数

| 操作数 | 类型                                      | 描述                               |
| ------ | ----------------------------------------- | ---------------------------------- |
| `r0`   | `[[U8; N]; num_keys]`                     | 验证密钥数组                       |
| `r1`   | `[[[Field; M]; num_instances]; num_keys]` | 每个电路每个密钥的嵌套公开输入数组 |
| `r2`   | `[U8; P]`                                 | 批量证明                           |
| `r3`   | `Boolean`                                 | 批量验证结果                       |

:::note
`snark.verify.batch` 仅在 `finalize` 块内有效。
:::
