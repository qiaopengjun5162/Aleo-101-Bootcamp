## 程序 (Program)

程序是驻留在 Aleo 区块链上某个程序 ID（Program ID）处的代码（其函数）和数据（其类型）的集合。程序的声明格式为 `program {name}.{network} { ... }`。程序的主体由花括号 `{}` 括起来。
```
import foo.aleo;

const FOO: u64 = 1u64;

struct Message {
    sender: address,
    object: u64,
}

fn compute(a: u64, b: u64) -> u64 {
    return a + b + FOO;
}

program hello.aleo {
    mapping account: address => u64;

    record Token {
        owner: address,
        amount: u64,
    }

    fn mint_public(
        public receiver: address,
        public amount: u64,
    ) -> (Token, Final) {
        let token: Token = Token { owner: receiver, amount };
        return (token, final {
            let current_amount: u64 = Mapping::get_or_use(account, receiver, 0u64);
            Mapping::set(account, receiver, current_amount + amount);
        });
    }
}
```
在 Leo 文件中，以下内容必须在程序的作用域（Scope）内声明：

映射与存储 (Mappings and Storage)

记录 (Records)

作为入口点的函数声明 (Entry point fn declarations)

在 Leo 文件中，以下内容必须在程序的作用域（Scope）外声明：

导入 (Imports)

常量 (Constants)

结构体 (Structs)

辅助函数定义 (Helper fn definitions)

final 函数定义 (final fn definitions)

接口定义 (interface definitions)

这些声明在当前的程序文件内是可以直接局部访问的。如果你需要使用来自其他 Leo 文件的声明，则必须将其导入。

## 导入 (Imports)

你可以导入已下载到 imports 目录中的依赖项。导入的声明格式为 `import {filename}.aleo;`。依赖解析器（Dependency resolver）会自动从网络或本地文件系统中拉取被导入的程序。
```
import foo.aleo; // Import all `foo.aleo` declarations into the `hello.aleo` program.

program hello.aleo { }
```
## 程序 ID (Program ID)

程序 ID 的声明格式为 `{name}.{network}`。

名称（name）的首字符必须是小写字母。名称只能包含小写字母、数字和下划线，且不能包含双下划线（__）或关键字 aleo。

目前，aleo 是唯一支持的网络域名（Network domain）。
```
program hello.aleo;     // 有效 (valid)

program Foo.aleo;       // 无效 (invalid)
program baR.aleo;       // 无效 (invalid)
program 0foo.aleo;      // 无效 (invalid)
program 0_foo.aleo;     // 无效 (invalid)
program _foo.aleo;      // 无效 (invalid)
program foo__bar.aleo;  // 无效 (invalid)
program aleo.aleo;      // 无效 (invalid)
```
