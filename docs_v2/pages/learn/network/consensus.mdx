# Consensus

Aleo 的共识协议称为 **AleoBFT** — 一种基于 DAG 的拜占庭容错（BFT）系统，受 Narwhal 内存池和 Bullshark 排序协议的启发。它提供**即时终局性**：一旦区块被提交，就是最终的且不可撤销。

## 工作原理

### DAG 结构

AleoBFT 将验证者通信组织成一个**有向无环图（DAG）**，构建在一系列轮次之上：

1. 每个验证者每轮广播**恰好一条消息**，引用前一轮至少 `n - f` 条消息（其中 `n` 是验证者总数，`f` 是拜占庭节点的最大数量）。
2. 验证者从对等节点收集签名以形成**批量证书** — 证明超级多数已确认该消息的密码学证明。
3. 这些证书构成 DAG 的顶点，边指向前一轮的引用证书。

### 领导者选择和提交

- **领导者**在每个偶数轮次选举产生。领导者的顶点称为**锚点（anchor）**。
- 当锚点从下一轮收到至少 `f + 1` 票时即被提交。
- 一旦锚点被提交，它引用的所有交易（以及 DAG 中可达的所有先前未提交锚点）都被排序并包含在区块中。

### 最终性

AleoBFT 实现即时最终性，因为 DAG 结构保证：如果任何诚实验证者提交了锚点，所有未来的锚点都将有一条路径指向至少一个为其投票的顶点。不存在分叉或链重组的可能性。

## 区块生产

当 DAG 提交时，验证者产生包含以下内容的区块：

- **交易（Transactions）** — 执行和部署交易及其 ZK 证明
- **批准（Ratifications）** — Coinbase 奖励分配
- **解决方案（Solutions）** — 累积的证明者难题解决方案
- **头部（Header）** — 先前状态根、交易根、元数据（高度、轮次、时间戳、供应量、证明目标）

区块需要获得**超过委员会投票权 2/3** 的批准（按质押权重计算）。

## 容错性

AleoBFT 容忍最多 `f` 个拜占庭验证者，其中 `f < n/3`。只要超过 2/3 的委员会（按质押权重计算）是诚实的，网络就会继续产生区块并维护共识。

## 时代（Epochs）

网络被划分为 **360 个区块的时代**。在每个时代开始时，从先前的区块哈希确定性地生成 `epoch_hash`。此哈希用于：

- **EpochProgram** — 证明者在该时代必须解决的随机抽样程序
- 难题尝试的输入随机化，防止预计算

该时代内的所有难题解决方案必须引用当前的 `epoch_hash`。
