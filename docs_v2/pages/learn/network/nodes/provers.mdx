# Provers

证明者通过解决**合成难题**（一种简洁工作证明，PoSW）为 Aleo 网络贡献计算能力。他们获得与其计算贡献成比例的 coinbase 奖励。

> **注意**：证明者不参与共识或产生区块。

## 合成难题

每个时代（360 个区块），网络生成一个新的 **EpochProgram** — 从时代哈希派生的随机抽样程序。证明者必须合成该程序以产生有效解决方案。

每次尝试的过程：

1. 从证明者的地址、时代哈希和计数器构造 nonce
2. 使用 nonce 作为种子采样随机输入
3. 使用这些输入为 EpochProgram 合成 R1CS 电路
4. 计算电路赋值的 Merkle 根
5. 将 Merkle 根转换为 `proof_target`
6. 如果目标满足当前的 `proof_target` 阈值，则提交解决方案；否则，递增计数器并重试

由于 EpochProgram 每个时代都会变化，并且每次尝试的输入都是随机的，因此解决方案无法预计算。

## 奖励

证明者获得 **coinbase 奖励的 2/3**。个人奖励计算公式为：

```
个人奖励 = 难题奖励 × (个人证明目标 / 组合证明目标)
```

这是一个按比例分配的模型 — 不是赢家通吃。每个有效解决方案都能获得奖励份额。

Coinbase 奖励在 10 年内逐渐减少，在第 9 年达到固定最小值。

## 质押要求

根据 "ARC-46协议"，证明者必须质押 Aleo credits 才能提交解决方案。所需质押在 24 个月内每季度增加，从 100,000 增加到 2,500,000 credits。完整时间表请参见"Run a Prover"部分。

## 目标重定向

网络使用 ASERT（绝对调度指数上升目标）算法动态调整 `proof_target` 和 `coinbase_target`，目标平均锚定时间为 25 秒。当累积证明目标达到 coinbase 目标的一半时，重新计算目标。

有关实际设置说明，请参阅 "Run a Prover"。
