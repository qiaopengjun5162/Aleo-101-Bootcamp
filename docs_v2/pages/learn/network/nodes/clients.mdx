# Clients

客户端提供区块链数据并充当用户与网络之间的接口。它们不参与共识、产生区块或解决难题。

## 角色

客户端负责：

- **中继交易** — 从用户接收交易并将其转发给验证者
- **提供区块链数据** — 通过 REST API 提供区块、交易和映射数据
- **引导对等节点** — 作为新节点加入网络的入口点

## 客户端类型

### Core 客户端

直接连接到共识网络上的验证者节点，并在流量到达验证者之前验证网络流量。Core 客户端**不**暴露 REST API，通常用作内部基础设施。

### Outer 客户端

连接到 P2P 网络上的其他客户端和证明者，作为面向公众的入口点。Outer 客户端在**端口 3030** 上暴露 **REST API**，用于查询区块链数据 — 区块、交易、程序状态和 映射值。

## 何时运行客户端

- **应用开发者** — 如果您的应用程序需要查询区块链状态或提交交易，Outer 客户端可以让您直接访问，而无需依赖第三方基础设施
- **基础设施提供商** — 运行客户端有助于在整个网络中分散数据可用性
- **验证者** — Core 客户端可以充当公共网络和验证者节点之间的缓冲区

有关实际设置说明，请参阅"Run a Client"。
