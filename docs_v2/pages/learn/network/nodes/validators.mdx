# Validators

验证者是产生区块、验证零知识证明并参与 AleoBFT 共识的节点。它们是 Aleo 网络的骨干。

## 角色

验证者负责：

- **验证 ZK 证明** — 在不查看底层数据的情况下检查转换是否正确执行
- **执行异步函数** — 在证明验证后运行链上逻辑（映射更新）
- **产生区块** — 参与 AleoBFT 共识以提交交易
- **维护状态** — 存储账本、映射和加密记录

## 委员会

活跃的验证者集合称为**委员会**，上限为 **200 个验证者**。成员资格由总质押量决定 — 质押量排名前 200 的验证者被纳入。委员会内的投票权与质押量成正比。

验证者必须满足以下要求才能加入委员会：

| 要求 | 值 |
|------|-----|
| 总质押量（自绑定 + 委托） | 10,000,000 Aleo Credits |
| 自绑定最低限额 | 100 Aleo Credits |

如果验证者的总质押量低于 1000 万 credits 或自绑定低于 100 credits，他们将被移除出委员会。

## 奖励

验证者获得 **coinbase 奖励的 1/3** 作为区块奖励，按其在委员会中的质押比例分配。验证者还可以设置**佣金率**（0-100%），决定他们保留委托者奖励的百分比。

## 配置

作为验证者绑定时，操作者设置：

- **佣金率** — 验证者保留的委托者奖励百分比
- **提款地址** — 解绑质押返回的地址
- **开放/关闭状态** — 是否接受新的委托

## 质押

Aleo 上的质押通过 `credits.aleo` 程序原生处理。用户可以将 credits 委托给验证者以参与网络安全并获得区块奖励份额。

### 关键参数

| 参数 | 值 |
|------|-----|
| 委托最低限额 | 10,000 credits |
| 最低绑定余额 | 10,000 credits |
| 解绑期 | 360 个区块 |
| 最大委托者数量（全网） | 100,000 |

### 委托

任何用户都可以使用 `bond_public` 函数将 credits 委托给验证者。委托的质押计入验证者的总质押量，用于委员会成员资格和投票权。

- 委托者自动获得与其质押比例对应的区块奖励份额，扣除验证者的佣金
- 验证者必须**开放**接受新委托，`bond_public` 才能成功
- 最低委托金额为 10,000 credits

### 解绑

质押者可以使用 `unbond_public` 解绑部分或全部质押。这将启动 **360 个区块的冷却期**（约 2.5 小时，按 25 秒区块计算）。冷却期结束后，任何人都可以调用 `claim_unbond_public` 将解绑的 credits 转移到质押者的提款地址。

- 允许部分解绑，但如果剩余绑定余额低于 10,000 credits，将触发完全解绑
- 如果验证者的总质押量低于 1000 万 credits，他们将被移除出委员会

### 提款地址

每个质押者都有一个**提款地址** — 解绑 credits 的目的地。这在绑定时设置，可以与质押地址不同，允许操作者分离运营和托管事务。

有关实际设置说明，请参阅"Run a Validator"。有关委托质押指南，请参阅"Staking"。
