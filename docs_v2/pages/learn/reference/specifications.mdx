# Specifications

## 导语
在区块链技术的语境中，规范是对系统架构、组件和操作的详细描述。它们充当开发者和工程师有效理解和实现系统的蓝图。对于 Aleo 而言，规范主要聚焦于两个领域：共识机制和虚拟机。

## AleoBFT
AleoBFT 是驱动 Aleo 区块链的共识算法，旨在确保验证者之间达成安全可靠的共识。AleoBFT 基于 Narwhal 和 Bullshark 的概念构建，并针对动态验证者委员会和权益质押参与进行了优化。在 AleoBFT 中，验证者收集交易并提交提案，这些提案随后由其他验证者背书以形成密码学证书。这些证书被组织成有向无环图（DAG），从而实现交易的高效排序并防止分叉，确保共识过程稳健且可扩展。

## AleoBFT 同步
当验证者（重新）加入网络时，它通过定期 ping 与对等节点交换紧凑的"检查点"和"最近"区块摘要，从而迅速了解自己所缺失的内容。通过比较这些定位器，它检测本地副本中的任何缺口，然后向轮换的对等节点集合分批请求缺失的区块。每个返回的区块都会与多个对等节点进行交叉校验，其内嵌的子 DAG 数据会被用于重建最近的证书图。一旦填补了所有缺口，该验证者即完成同步，可以立即参与新区块的提案和背书。

## AleoVM
Aleo 虚拟机（AVM）是 Aleo 区块链中不可或缺的计算平台，专为运行隐私优先的应用而设计。它以栈式机器的形式运行，执行排队的指令。其主要功能是为函数中的每条指令构造以秩-1约束系统（R1CS）描述的算术电路。构造完 R1CS 后，使用 Marlin 算法的一种变体——Varuna 来生成相应的证明。这种方法通过利用通用且可更新的结构化参考字符串（SRS），实现了对任意计算的简洁验证。AVM 对于维护 Aleo 网络上的隐私性和可扩展性至关重要。它确保交易和合约在数据保密的前提下执行，使用 zk-SNARK 来验证计算的正确性，而无需泄露输入或内部状态。
