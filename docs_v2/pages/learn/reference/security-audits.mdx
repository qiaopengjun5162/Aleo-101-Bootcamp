# Security Audits
## 导语
snarkVM 栈（包括 AleoVM 组件）以及更广泛的 Aleo 技术栈已经过了第三方安全审查和审计。本页汇总了官方公告和公开的审计报告。博客文章偏重宏观概述；审计报告才是关于发现的问题、严重程度和修复状态的权威来源。

## 博客文章与公告
### 审计公告与摘要

- **[Aleo 完成 snarkOS 与 snarkVM 安全审计](https://aleo.org/post/aleo-completes-security-audits-of-snarkos-and-snarkvm/)** — Aleo 基金会公告，总结了针对 Aleo 节点（snarkOS）和虚拟机/密码学栈（snarkVM）的审计结果。
- **[Aleo 合成器审计](https://blog.zksecurity.xyz/posts/aleo-synthesizer/)** — zkSecurity 博客文章，总结了其对 Aleo 合成器的审计。

### 安全保障工程

- **[约束系统的形式化验证](https://provable.com/blog/formal-verification-of-constraint-systems)** — Provable 对编译阶段和约束系统组件进行形式化验证的工作。
- **[庆祝 AleoBFT 形式化验证里程碑](https://provable.com/blog/fuzzing-the-Aleo-VM)** — Provable 在 ACL2 模型中对 AleoBFT 各项属性进行形式化建模与证明的工作。
- **[模糊测试 Aleo VM：实现绝对安全性的两种方法](https://provable.com/blog/fuzzing-the-Aleo-VM)** — Provable 为加固 AleoVM/snarkVM 所制定的模糊测试策略。

### 公开审计报告

- **[zkSecurity 审计报告](https://reports.zksecurity.xyz/)** — zkSecurity 的公开审计报告，包括与 Aleo 相关的组件。
