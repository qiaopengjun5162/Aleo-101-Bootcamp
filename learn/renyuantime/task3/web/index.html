Aleo · Zero-Knowledge Credential

ZK 私密资质通行证

证明 “信用分 ≥ 阈值” 为真，却 不暴露真实分数，也 无需信任平台方。 所有计算均来自 Aleo / Leo 的真实零知识执行，三方角色一屏闭环。

本地真实执行 · leo run 隐私要点：验证方永远看不到真实分数 后端：检测中…
1 Issuer / 发证方

签发私密凭证

征信机构把用户的真实分数封装进私有 record，链上只存加密承诺。

尚未签发。点击「签发凭证」调用 leo run issue_credential。
2 Holder / 持证人

生成「达标」证明

持证人用自己的凭证证明「分数 ≥ 阈值」，证明里不含真实数值。

尚未验证。点击「生成达标证明」调用 leo run verify_threshold。
3 Verifier / 验证方

核验结果（零数值视角）

平台只看到「通过 / 未通过」与公开计数，永远拿不到真实分数。

等待持证人提交证明…
